TECHNICAL KNOWLEDGE
═══════════════════════════════════════════════════════════════════════════

Q6: What is the difference between frontend and backend?
A:  Frontend: What users see and interact with (UI/UX)
    ✓ Technologies: HTML, CSS, JavaScript, React, Vue
    ✓ Focus: User interface, user experience
    ✓ Runs on: Browser
    
    Backend: Server-side logic and data management
    ✓ Technologies: Node.js, Python, Java, MongoDB, databases
    ✓ Focus: Business logic, APIs, databases
    ✓ Runs on: Server
    
    Full-stack: Developer works on both

---

Q7: What technologies do you know?
A:  Frontend:
    ✓ HTML, CSS, JavaScript
    ✓ React / Vue / Angular
    ✓ Bootstrap, Tailwind CSS
    
    Backend:
    ✓ Node.js & Express
    ✓ MongoDB / MySQL
    ✓ REST APIs
    
    Tools:
    ✓ Git, GitHub
    ✓ Postman
    ✓ VS Code

---

Q8: What is MVC architecture?
A:  Model-View-Controller pattern:
    
    Model: Data and business logic
    View: User interface (frontend)
    Controller: Handles requests and updates model/view
    
    Example:
    ✓ User submits form → Controller receives request
    ✓ Controller queries Model (database)
    ✓ Model returns data
    ✓ Controller passes to View
    ✓ View displays to user

---

Q9: What is REST API?
A:  REST = Representational State Transfer
    
    Uses HTTP methods:
    ✓ GET - Retrieve data
    ✓ POST - Create data
    ✓ PUT - Update data
    ✓ DELETE - Delete data
    
    Example:
    GET /api/posts → Get all posts
    POST /api/posts → Create new post
    PUT /api/posts/1 → Update post 1
    DELETE /api/posts/1 → Delete post 1

---

Q10: What's the difference between SQL and NoSQL?
A:  SQL (Relational):
    ✓ Table-based structure
    ✓ Fixed schema
    ✓ Examples: MySQL, PostgreSQL
    
    NoSQL (Document-based):
    ✓ Document/collection-based
    ✓ Flexible schema
    ✓ Examples: MongoDB, Firebase
    
    When to use:
    ✓ SQL: Structured data, complex queries
    ✓ NoSQL: Flexible data, scalability

Q25: What is React?

    with flexible schema."

Q28: What is Git?
A:  "Git is version control system used to track code changes and collaborate 
    with team members."

Q29: What is an API?
A:  "API (Application Programming Interface) is a set of rules for 
    applications to communicate with each other."

Q30: What's the difference between var, let, and const?
A:  "var: Function-scoped, can be redeclared
    let: Block-scoped, can't be redeclared
    const: Block-scoped, can't be redeclared or reassigned"

🎯 INTERVIEW TIPS FOR SUCCESS
═══════════════════════════════════════════════════════════════════════════

Before Interview:
✓ Research the company (mission, products, tech stack)
✓ Review your projects and be ready to explain them
✓ Practice common questions aloud
✓ Prepare examples with numbers and metrics
✓ Know your resume inside out
✓ Have GitHub and portfolio links ready
✓ Prepare 3-5 questions to ask
✓ Get good sleep night before

During Interview:
✓ Arrive on time (or join early if online)
✓ Dress professionally
✓ Maintain eye contact
✓ Speak clearly and confidently
✓ Be honest - don't exaggerate skills
✓ Listen carefully to questions
✓ Give specific examples with numbers
✓ Show enthusiasm and passion
✓ Ask follow-up questions
✓ Write down important points
✓ Smile and be friendly

Answers Should:
✓ Be specific with examples (not generic)
✓ Show your thought process
✓ Demonstrate learning and growth
✓ Include what you learned from mistakes
✓ Be honest about what you don't know
✓ Show interest in the company/role

Red Flags to Avoid:
✗ Speaking negatively about previous company
✗ Exaggerating your skills
✗ Talking too much or too little
✗ Not listening to the question
✗ Being unprepared
✗ Appearing disinterested
✗ Checking your phone
✗ Being arrogant or defensive

═══════════════════════════════════════════════════════════════════════════
